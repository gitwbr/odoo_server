# Tasks: dtsc 概覽畫面

- [ ] 1.1 新增後台菜單 `概覽畫面`，放在 `報表` 後面。
- [ ] 1.2 建立概覽頁 action/view 或 client action 空殼。
- [ ] 1.3 空殼頁顯示科技感版面框架：KPI 區、圖表區、排行區、異常提醒區。
- [ ] 1.4 套用 `dtsc` 後台權限，未授權使用者不可進入。

## Phase 2: 大圖訂單、出貨單、應收單
- [ ] 2.1 建立概覽統計服務，集中封裝 read_group/search_count 查詢。
- [ ] 2.2 大圖訂單 KPI：今日/本月訂單數、狀態數、類別數、未稅金額、含稅金額、總才數。
- [ ] 2.3 出貨 KPI：今日出貨、逾期未出貨、已生成出貨單、未生成出貨單、出貨數量與才數。
- [ ] 2.4 應收 KPI：已轉應收、價格已審未轉應收、應收金額、帳單月分布。
- [ ] 2.5 加入對應清單跳轉，點擊 KPI 可進入過濾後列表。

## Phase 3: 採購、應付、明星產品、材料消耗最快
- [ ] 3.1 採購 KPI：採購單數、採購金額、待確認/待收貨/已完成狀態。
- [ ] 3.2 應付 KPI：供應商帳單數、未付/已付、應付金額、供應商排行。
- [ ] 3.3 明星產品：依 `dtsc.checkoutline` 統計產品的金額、才數、數量排行。
- [ ] 3.4 材料消耗最快：第一版依 `dtsc.checkoutline` 統計產品/材料使用量與才數排行。
- [ ] 3.5 在畫面上明確標示材料消耗第一版為「依訂單行使用量」，不是庫存扣料。
